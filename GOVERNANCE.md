# Governance and Genesis Risk

HAP v1 is a founder-authored genesis specification. First-mover influence is real and is not disguised as decentralisation.

## No protocol administrator

There is no upgrade key, mandatory server, canonical website, token vote, or founder veto. Existing versioned objects retain their original interpretation.

## Changes

Changes are proposed publicly with:

- a versioned specification;
- rationale and rejected alternatives;
- conformance vectors;
- migration and compatibility analysis;
- security and privacy implications.

Unknown critical versions are rejected. Implementations may support multiple versions concurrently.

## Funding independence

Voluntary project funding, grants, sponsorships, and operator service fees do not create protocol authority. Funders receive no validity privilege, truth status, veto, mandatory fee, or ownership of HAP.

The official v1.0.0 source release fixes a genesis Bitcoin contribution address into its versioned release metadata. This is an attribution property of that release, not a consensus rule or ownership mechanism. Future maintainers may publish a separately signed replacement destination, and forks may use different funding, but any change necessarily produces a different source revision, manifest identifier, and release artifact. Compromise or disappearance of any funding destination must not interrupt publication, verification, preservation, or independent development. Material project funds should move toward transparent accounting and multisignature custody as independent maintainers emerge.

## Path away from founder dependence

The project should earn legitimacy through independent adoption by:

- encouraging a second independently written validator;
- maintaining public conformance and adversarial vectors;
- using reproducible builds;
- adding independent maintainers;
- mirroring repositories and releases;
- avoiding trademark, domain, or repository dependencies for technical participation.

GitHub is a convenient publication channel, not the protocol authority.
